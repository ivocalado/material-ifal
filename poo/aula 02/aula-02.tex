\begin{frame}{Como Java funciona?}
\begin{block}{O que são \textit{bytecodes}?}
\textit{O código de um programa de computador escrito na linguagem Java é compilado para uma forma intermediária de código denominada bytecode, que é interpretada pelas Máquinas Virtuais Java (JVMs). É essa característica que faz com que os os programas Java sejam independentes de plataforma, executando em qualquer sistema que possua uma JVM.}
\end{block}

 
\end{frame}

\subsection{Hello World}
\begin{frame}[allowframebreaks]{Hello World}
Criar novo projeto Java
\begin{enumerate}
 \item Abrir o Eclipse
 \item Criar novo projeto Java
 \begin{enumerate}
  \item Menu \textit{File}
  \item Opção \textit{New}
  \item Selecionar opção \textit{Java Project}
 \end{enumerate}
 \item Informar o nome do projeto e selecionar \textit{Finish}
 

\end{enumerate}

Criar classe \textit{Main}
\begin{enumerate}
 \item Menu \textit{File} -$>$ New
 \item Selecionar opção \textit{Class} 
 \item Informar o nome da nova classe (``App'') no campo \textit{Name} e selecionar \textit{Finish}
\end{enumerate}

\begin{block}{Que nome utilizar para a classe? (Importante)}
\begin{itemize}
 \item Toda classe Java deve iniciar com letra Maiúscula (Pessoa, Funcionario, Carro, etc)
 \item Qualquer aplicação Java deve ter pelo menos uma classe para conter o método \textbf{main}
\end{itemize}


\end{block}

\end{frame}

\begin{frame}[fragile,allowframebreaks]{O método main}
\begin{itemize}
 \item Para que uma aplicação Java possa ter início \textbf{alguma classe} deve ter o método \textbf{main}
 \item O método \textbf{main} representa o \textit{ponto de partida} a partir do qual a JVM irá iniciar a aplicação Java
\end{itemize}

 \begin{lstlisting}
  public class App {
	public static void main(String[] args) {
		//A JVM inicia a execução da aplicação Java a partir deste ponto
	}
  }
 \end{lstlisting}
 
 \begin{itemize}
  \item A assinatura do método \textbf{main} é fixa
 \end{itemize}
\begin{block}{O que é a assinatura de um método?}
 Um método é definido pelo seu nome e pelo conjunto de entradas e pelo tipo de saída. A tal estrutura damos o nome de \textbf{assinatura de método}
\end{block}

\begin{block}{Seria possível criar uma função fora de uma classe?}
Não! Diferentemente de Python e outras linguagens não é possível definir uma função fora de uma classe. Neste sentido, qualquer aplicação Java deve conter pelo menos uma classe que irá conter o método \textbf{main}
\end{block}


\end{frame}

\begin{frame}[fragile]{Como imprimir uma mensagem na tela?}
 \begin{lstlisting}
public class Main {
	public static void main(String[] args) {
		System.out.println("Olá mundo!");
	}
}
 \end{lstlisting}
\begin{itemize}
 \item O comando \textbf{System.out.println} imprime na saída padrão a mensagem passada por parâmetro inserindo o caracter de nova linha ao final da mensagem
 \item Uma possibilidade seria utilizar o comando \textbf{System.out.print} em que não é inserido o caracter de nova linha ao fim da mensagem
 \item \textbf{Comandos devem sempre terminar com ;}
\end{itemize}

\end{frame}

\begin{frame}{Executando nosso primeiro programa}
\begin{enumerate}
 \item Clicar com o botão direito no arquivo Java que contém o método \textbf{main}
 \item Ir na opção \textit{Run as...} e selecionar a opção \textit{Java application}
\end{enumerate}
\begin{block}{Como compilar e executar o programa a partir da linha de comando?}
 \begin{enumerate}
  \item Para compilar deve-se posicionar o console na pasta onde contem o arquivo java e executar o comando \textit{javac App.java}, que irá gerar o arquivo \textbf{App.class}
  \item Para executar deve ser executado o comando \textit{java App} (sem a extensão \textbf{.class})
 \end{enumerate}

\end{block}
\end{frame}


\begin{frame}[fragile]{O que é importante sabermos até o momento?}
\begin{itemize}
 \item Toda aplicação deve conter pelo menos uma classe Java que deverá possuir um método \textbf{main}
 \item Toda aplicação deve ter pelo menos um método \textbf{main} que será responsável por iniciar a aplicação
 \item A assinatura do método \textbf{main} é fixa
\end{itemize}
\begin{block}{}
\begin{itemize}
\item Palavras-chave \textbf{public, static, void, String} são importantes mas serão melhor detalhadas à frente
\end{itemize}
\end{block}


\begin{lstlisting}

\section{Sintaxe da linguagem}
\subsection{Elementos básicos da linguagem}
\begin{frame}{Variáveis}
Assim como em outras linguagens de programação, variáveis \textbf{representam espaços de memótia utilizados para armazenar dados}
 
\begin{block}{O que há de diferente para Python?}
\begin{itemize}
 \item Em Python, ao declarar a variável, você não define seu tipo. Além disso, você pode atribuir valores de qualquer tipo que a variável se ``transforma'' no tipo de destino
 \item Em Java cada variável tem um tipo específico definido no momento da declaração. \textbf{Na grande maioria das vezes não é possível fazer a conversão automática entre os tipos}
\end{itemize}

\end{block}

\end{frame}

\begin{frame}[fragile]{Como declarar uma variável?}
Para definirmos uma variável temos duas abordagens possíveis:
\begin{lstlisting}
 tipo nome;
\end{lstlisting}
Ou

\begin{lstlisting}
 tipo nome = valor;
\end{lstlisting}

Por exemplo:
 \begin{lstlisting}
public class Main {
	public static void main(String[] args) {
		int i;
		int j = 0;
	}
}
 \end{lstlisting}

\end{frame}

\begin{frame}{Quais são os tipos primitivos possíveis?}
       \begin{center}
         \begin{tabular}{|c|c|p{5cm}|}
         \hline
	 \textbf{Tipo} & \textbf{Valores possíveis} & \textbf{Descrição} \\ \hline
	 boolean  & true $|$ false& representa valores booleanos\\ \hline
	 short  & -32768 $\leq$ x $\leq$ 32767& representa valores inteiros \\ \hline
	 int  & -2$^{31}$ $\leq$ x $\leq$ 2$^{31}$ - 1 & representa valores inteiros\\ \hline
	 long  & -2$^{63}$ $\leq$ x $\leq$ 2$^{63}$ - 1 & representa valores inteiros\\ \hline
	 float  & 1.456, -0.00021, 5.79, ...& representa valores em \\ \hline
	 double  & 1.456, -0.00021, 5.79, ... & representa valores em ponto flutuante\\ \hline
	 char  & 'a', 'b', 'C', ...& representa um caracter\\ \hline
         \end{tabular}
       \end{center}
       
\end{frame}

\begin{frame}[fragile]{Quais são os nomes possíveis de variáveis?}
\begin{itemize}
 \item Iniciadas com letra maiúscula ou minúscula, e \_
\end{itemize}
\begin{block}{Existe alguma recomendação sobre como declarar uma variável?}
 Sim! 
 \begin{itemize}
  \item Recomenda-se que qualquer variável seja iniciada com letra minúscula
  \item Nomes compostos devem seguir a regra \textit{Camel Case} (a cada nova palavra coloca-se em maiúscula)
 \end{itemize}

\end{block}

Exemplo
\begin{lstlisting}
 double gratificacaoNatalina;
\end{lstlisting}


\end{frame}

 \begin{itemize}
  \item Não podem ser utilizados como identificadores de variáveis palavras-chave da linguagem (a IDE é sua amiga, ela te alerta!)

\subsection{Comandos de Entrada e Saída}
\begin{frame}[fragile]{Comandos de entrada}
\begin{itemize}
 \item Para lermos o valor de alguma variável fazemos uso da classe \textbf{Scanner}
 \item O objeto Scanner vai ler da entrada padrão e ``entregar'' o tipo desejado deixando o restante da entrada disponível para novas leituras
\end{itemize}


 
 \begin{lstlisting}
import java.util.Scanner;
public class Main {
   public static void main(String[] args) {
      Scanner scanner = new Scanner(System.in);
      int i = scanner.nextInt();
      int j = scanner.nextFloat();      
      scanner.close(); //sempre deve ser invocado ao final do programa para fechar o fluxo
   }
}

 \end{lstlisting}

\end{frame}

\end{itemize}

\begin{lstlisting}

\end{lstlisting}

\begin{frame}{While e Do-While}
\begin{figure}
 \centering
 \includegraphics[scale=0.35]{./imagens/while-sintaxe.png}
 % while-sintaxe.png: 0x0 pixel, 300dpi, 0.00x0.00 cm, bb=
\end{figure}

\begin{itemize}
 \item No While e no do-while é passado apenas um parâmetro que indica que será avaliada para determinar se o loop deve continuar
 \item O While \textbf{testa} antes de \textbf{executar}
 \item O do-while \textbf{executa} antes de \textbf{testar}
\end{itemize}


\end{frame}

\begin{frame}[fragile]{Exemplo 1: while}
 \begin{lstlisting}
int x = 0;
while (x < 10) {
	System.out.println(x);
	x++;
}
 \end{lstlisting}
 \begin{itemize}
  \item O comando while testa a condição antes de executar o bloco de comandos
  \item Ele continua a execução do bloco até que a condição se torne \textbf{falsa}
 \end{itemize}  

\end{frame}

\begin{frame}[fragile]{Exemplo 2: do-while}
\begin{lstlisting}
int x = 0;
do {
	System.out.println(x);
	x++;
} while (x < 10);
\end{lstlisting}
 \begin{itemize}
  \item O comando do-while executa o bloco inicialmente para, só então, testar se deve continuar o loop
  \item Ele continua a execução do bloco até que a condição se torne \textbf{falsa}
 \end{itemize}  

\end{frame}

\begin{frame}{\textit{break} e \textit{continue}}
\begin{itemize}
 \item \textbf{break}: interrompe a execução do bloco de repetição
 \begin{itemize}
  \item Continua com a próxima instrução, logo após o bloco
 \end{itemize}
 \item \textbf{continue}: interrompe a execução da iteração
 \begin{itemize}
  \item Testa a condição e reinicia o bloco com a próxima iteração
 \end{itemize}


\end{itemize}

\begin{frame}[fragile]{Arrays}
 \begin{itemize}
  \item Em Java é possível criar arrays (vetores) para o armazenamento de mais vários do mesmo tipo

  int[] vetor = new int[5];
  vetor[0] = 10;
  vetor[1] = 20;
  vetor[5] = 5; //Erro!
  int k = vetor.length; // k == 5
 \end{lstlisting}
